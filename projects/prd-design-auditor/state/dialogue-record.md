## 2026-03-10T15:30:00Z - Token 优化战役与架构边界确立
*记录人：前线 AI + 指挥官 Gemini*

- **决策 1**：采用"触发器路由模式"重构 Commands/ 架构
  - **依据**：Claude Code 无条件预加载 `~/.claude/commands/*.md`，导致 47.7KB 详细文档全量加载
  - **实施**：将详细内容下沉至 `.skills/`，原文件截断至 200 字节路由
  - **收益**：Commands/ 大小减少 81%，后续交互 Token 减少 95%
  - **责任人**：前线 AI

- **决策 2**：停止后续 Token 优化，设定架构红线
  - **依据**：边际收益递减法则 - Memory/ 精简需 30 分钟仅节省 4K tokens，且可能导致系统失忆
  - **红线约束**：
    - ⛔ **严禁继续深挖 Memory/Common 导致母舰失忆**
    - ✅ 接受现状：38K 首次加载在可接受范围内
  - **责任人**：指挥官 Gemini

- **决策 3**：采用软链接中央配置架构
  - **依据**：需要管理多个项目的共享配置，避免重复维护
  - **实施**：
    - 详细内容统一放在 `ai-coding-standards/.skills/`
    - 项目通过软链接按需引入
  - **收益**：单点维护，全局生效；项目按需链接，避免无关污染

- **关键数据验证**：
  - 优化前：Commands/ 47.7KB，后续交互 39,138 tokens
  - 优化后：Commands/ 9.2KB，后续交互 1,901 tokens
  - 首次加载：38K → 38K（几乎无变化）
